HTML

HTML Basics

  • HTML stands for Hypertext Markup Language (HTML), which is used to give the content structure
  • HTML is one of the three cornerstone languages used in web development: HTML, CSS, and JS
  • An element is a type of content on a webpage. The following element types are commonly used to add content to pages:
    • Text elements
    • Image elements
    • Header elements
    • List elements
  • In addition to elements that can be seen on the page, elements are also used to divide the content on the page and to describe the content to browsers and other developers
  • Here are some common element types that are used to divide and describe a page:
    • Meta-related elements
    • Content division elements
    • Semantic elements
  • As developers, one of the first things we must think about when developing a webpage is building a sturdy "frame" using HTML
  • First, we use HTML to build a sturdy frame, using HTML elements as our frame’s building blocks. Next, we add CSS to define the style. Finally, we add JavaScript to our HTML structure to make the website interactive
  • These HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets < >. These tags contain everything that we want to include in the element. This can include content and even other nested elements
    • Remember, again, to almost always have an opening tag and closing tag. For example, for your body, you would open with <body> and, after all of your content, would close with </body>. The /[element name] represents a closing tag
    • Some elements (primarily formatting elements) do not require a closing tag, such as the line break element <br>
  • The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference
    • An element refers to the entire element from opening tag to closing tag
    • A tag refers only to what appears inside the angle brackets
  • Most element tags can take attributes, or modifiers that can define or change how an element is rendered or handled in the browser. There will be several examples below, but the attribute will always come after the element name in the tag
    • An example would be <a href="http://someurl.com">. Here, the element is a or anchor, and the attribute is href which modifies the anchor element to have a URL to create a hyperlink
  • Start your HTML code with <!DOCTYPE html> to let the browser know what type of document it is
  • Next, use <html> to know when our HTML document begins
    • Add a lang attribute to let the browser know what language this site/page is in. For our example, we use <html lang=”en”> to tell the browser the site is in English
  • Two elements are especially important to understanding how the structure works: the <head> and <body> elements
    • The <head> element contains information, or metadata, that helps the browser to render the page correctly. Elements included within the <head> are not visible when a user looks at a webpage
    • The <body> element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element
  • Use <!--comment--> to make comments in your HTML file

HEAD Element Basics

  • The <head> element is a container for metadata. <meta> elements contain information about the page that is used by the browser. The user won’t see information in the <head> or <meta> elements
  • The <head> element contains the metadata for our web application. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance
  • Inside the <head> element, we can use the following metadata tags:
    • The <title> element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO)
    • The <style> element defines the CSS styles associated with a website
    • The <base> element defines the base URL for a webpage
    • The <link> element connects an external resource to the HTML document
    • The <meta> element defines metadata such as the character set, description, keywords, author, and viewport
    • The <script> element embeds client-side scripts in the HTML document

BODY Element Basics

  • The body of an HTML document includes the content that users see when they look at the webpage in a browser
  • Elements that are nested inside other elements are called child elements
  • The <body> element is an example of a semantic HTML element
    • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
    • When possible, it is important to use semantic HTML elements, because they make your HTML easier to read by other developers and the browser
    • Semantic HTML also plays an important role in making sure your webpage is accessible to the widest variety of users, including those who use assistive technologies such as screen readers
  • Not all elements are semantic, though. It is still common to see non-semantic elements such as <div> to create a division in the page, or <span> to create an inline division
    • Sometimes these are simply the most efficient solution to a structural problem the developer is trying to solve. The most important thing to keep in mind is that it is best practice to use semantic elements whenever possible, especially when the content is visible to users
  • The <h1>, <h2>, <h3>, <h4>, <h5> and <h6> elements represent the level of heading a given text block represents
    • Order matters! When using headings, it is important that we use heading tags in order from <h1> to <h6> to make the page accessible to users who access the page using screen readers or other devices
    • In addition, the biggest heading, <h1>, should only appear once on each page
  • Here is a list of some child elements inside the <body> tag
    • The <p> element represents a paragraph or block of text. You'll use this element extensively to wrap most of the text on your webpages
    • The <ul>, <ol>, and <li> elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists
    • The <img> element contains information about images that are displayed on the webpage. The <img> element can contain various attributes; the src attribute is the most important because it defines the location of the image
    • In an <img src=”” alt=””> tag, the alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website
    • The <br> element creates a line of empty space, or a line break, between two blocks of content
    • For hyperlinks, the <a> element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link
  • The <footer> appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

CSS

CSS Basics

  • To add style to our webpage, we will use Cascading Style Sheets (CSS), which is a programming language used by developers to define how webpage content is presented to users
    • Whereas HTML strictly handles the structure and content of a webpage, CSS defines the colors, aesthetics, visual layout, and organization
    • We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline CSS
    • Used when a developer wants to style an element directly in the HTML file
    • Useful for quick and permanent changes, but less flexible than other types
    • An example could be <h1 style =”color:red;”>
  • Internal CSS style sheet
    • Used to embed an entire stylesheet directly in the HTML file using a <style> element in the <head> element
    • This internal style sheet contains CSS rules for the webpage in the head section of the HTML file
    • The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code
    • An example is: 
      <head>
  <style>
    h1 {
      color: red;
      } 
  </style>           
 </head>
      • This example would make all <h1> elements red for this HTML file
    • This method is totally acceptable, but most websites use the external CSS style sheet method
  • External CSS style sheet
    • Most common use of CSS style sheets
    • Preferred because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier
    • When using an external style sheet, we link the file using a <link> element placed in the HTML's <head>
    • An example is: 
      <head>
  <link rel=”stylesheet” href =”./assets/style.css” />
</head>
      • This example links (or includes) the CSS style sheet in our page which then implements all style properties to the linked elements in our page or site
    • Important: when using an external CSS sheet, always remember to link it in whatever pages you want to use it in (or a master header if using one) otherwise your styling won't appear at all!

CSS Rules & Terms

  • When defining a CSS rule, you will have a selector, declaration block, declarations, properties, and values
  • An example of a CSS rule is: div {color:blue; text-align: justify;}
    • In this example:
      • div acts as the selector
      • { } is the declaration block
      • color:blue; and text-align: justify; are declarations
      • color and text-align are properties
      • blue and justify are the values
  • The selector defines the element or attributes to which the rules, or declarations, will apply
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue; includes the CSS property color, and the value for that property, blue
  • For the color property, you can use semantic references such as blue or gray, but you can also use hex # or RGB numbers to define a color. Find a list of the HEX values here

CSS Selectors & Properties

  • Selectors can be used to define style for any visible HTML element
  • Properties are the "options" of style you can give to the content inside or outside the element
  • The padding property is often confused with margin, but it is not the same thing. The padding property adds space around the content inside an element. When you increase padding, you are actually making the entire element bigger! That is because the padding, while transparent, is located inside the element, whereas the margin is the transparent buffer outside an element
  • To understand this better, it is important to know that every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content. This is sometimes referred to as the CSS box model
  • Sometimes, we want to apply a rule to ALL the elements visible on the page. To do that, we use a special wild card symbol, or *
  • You can combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors. Rather than repeating a rule which would add more lines of code and repetition, two or more elements that will share the same styling can be grouped together. This helps us write Do Not Repeat Yourself (DRY) code
    • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase
  • text-align: center; is one way to center a title or heading on a webpage
  • The img selector assigns property values to any image element in the markup
  • To give an example of how to format your CSS rules, here is an example of the img selector: 
    img {
  display: block;
  height: 50px;
  width: 50px;
  margin-left: auto;
  margin-right: auto;
}
    • The display property is assigned with the block value, which assigns each image to start on a new line. The height and width are assigned number values, which will stay constant on the browsers. The margin values are assigned auto, which will cause the margins to adjust based on the display
  • Additional properties, to name a few, could be:
    • font-size which would define the size of the font (ex: font-size: 12px;)
    • font-family which would define the type of font (ex: font-family: monospace;)
    • border which would define a visible border to the element (ex: border: 1px solid black;)
    • background-color which would define the color of the background of the element (ex: background-color: blue;)
    • box-shadow which would define a drop-shadow for the box (ex: box-shadow: 5px 10px #888888;)

Adding Class Attributes

  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule
  • This would be used in a scenario where we would like to add styles to certain elements, but not all the elements
  • To achieve this, we would add a . in front of the selector in our stylesheet, and the selector would be a name we choose
    • An example of a selector would be .card
  • This would be implemented into your element using class="(selector name)"
    • A full example of this would be <section class="card">
    • Notice that the class attribute does not have the . in front of the selector name! The . is only used on the stylesheet, not the HTML code

Git

Git Basics & Git/GitHub Comparison

  • Git is the most commonly used version control software among developers and a tool that you will likely use on the job
  • Git is a version control system that lets you manage and keep track of each version of your project
    • Git is accessed through the command line (terminal on macOS/Linux systems, git bash on Windows) on our computer using the git command
      • Git does not work out of the box on any system and will require some degree of installation to function in your command line. The installation procedure varies by OS type
    • Git is used to save and track versions of your code. It is also used to initiate a repository (or repo), add our code to the repo, and view the different versions of our code, which are each stored in a unique commit
    • Git is a great tool for tracking our versions and code, but can only be used locally
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely
    • Using GitHub not only allows us to store our code, but also to easily collaborate with others
    • It is important to note that while GitHub is popular, there are other Git-based repository hosting services, such as Bitbucket and SourceForge. However, for this course, we will use GitHub, which is free, to share work and collaborate
    • GitHub will also act as your portfolio of code to show your skills to potential employers
  • Review Git command cheatsheet here
  • git status checks what branch we are currently on
  • Be sure to set up a global username and email using the following:
    • git config --global --replace-all user.name "username"
    • git config --global --replace-all user.email "email@email"

Repositories

  • Repositories are directories that contain all project files
  • Repositories can be made locally using git init, but we will creating repositories in GitHub and then cloning them to our local machine

Branches

  • Branches are used to make changes, additions, or deletions without affecting the main and/or deployed source code. It is always best practice to use a branch for any and all changes
  • There can be different types of branches:
    • Feature branch
      • When a developer is working on implementing or improving a single feature of the project
    • Develop branch
      • When a single or team of developers is working on a major feature or set of features that may drastically change the product
      • These branches often have “sub” feature branches for all these features to be worked on, and will be pulled to the develop branch first before being eventually deployed to the main branch
  • git checkout -b branch-name creates a new branch and switches to it
  • You should always use git pull origin main to ensure you are working with the current version of the main branch (if working with a remote repo) before you create a new branch and before you push commits to GitHub
  • Use git checkout main to switch back to your main branch once you are done working in your branch

Stage and Commit

  • To stage changed files in your branch:
    • git add -A OR git add [file]
    • use git restore [file] to remove files from stage
  • To commit your changes to the branch, use git commit -m "[message on what was changed]"
  • Every commit should be a unique change to be able to successfully and easily track and remedy bugs
  • Once you have committed, you should merge (if working locally and done with all feature changes) or push your commit (if working with GitHub or any other remote Git platform)

GitHub Basics

  • GitHub is the web interface for Git, a version control software, that allows multiple people to make separate changes to a program at the same time without getting in each others way
  • GitHub is also a great way to work on remotely deployed software without affecting the main deployment and/or user experience

GitHub terms

  • Repository or repo
    Every project on GitHub is a repository

  • Branch

    A way to write for the web using plain text that easily converts to HTML
  • Commit (changes)
    Save a new version of the file you are working on
    If you finish adding, editing, or deleting a file or single change, you should commit
    Adds a change to your branch
  • Pull request “PR”
    Asking for changes on a branch to merge into another branch
    Way to discuss and review your work with others
  • Merge
    When a PR is approved, changes are pulled in or merged into the parent or main branch
    Changes publish/deploy to the live site once this happens if merging into the main branch
  • Issue
    Way to report new tasks or content fixes
    Track progress on a project board
    Open an issue for a new task and close once done

Updating content

Creating a Repository

  • When creating a new repo:
    • We should usually include a readme.md file
    • We should usually include a .gitignore file
      • This allows us to have files in the repo that we do NOT want uploaded to the cloud (i.e., ssh keys or login info)
    • We should almost always select a license
      • MIT license is the most open/unrestricted license we can choose and will be the one we choose in this course
  • When naming a repository, it's important to remember the following conventions:
    • Be unique and descriptive for each repository name
    • Use all lowercase letters
    • Use dashes; we can't have spaces in our repository names, so we use dashes instead
    • Be consistent! Our GitHub account will follow us throughout our development career, so show it some love
  • Add a description to your repository. Be concise and clear with your project descriptions
  • Use git clone [GitHub provided URL] to download your repo to your local machine
    • Make sure you are in the directory that you want to clone it to
    • Use git remote -v to track what URLs you need to use to fetch or push files
  • Create GitHub Issues using Markdown
    • GitHub offers a task management tool called GitHub issues, which gives developers and teams an easy way to keep track of key tasks and progress. Think of it as a built-in to-do list right inside our GitHub repository
    • GitHub issues can also help teams to do the following:
      • Track, organize, and communicate ideas, tasks, bugs, and enhancements
      • Identify bottlenecks or blockers that could prevent work progress
      • Avoid duplicating work by assigning tasks or issues

Pulling Branch Changes, Pull Requests, and Merges

  • To pull your branch changes from your local machine in Git
    • First, double check you are working with the latest version of the code with git pull origin main
    • Next, push your changes for your branch with git push origin [branch name]
    • If you are done with the branch, you should pull and merge in GitHub, and then delete the branch on both GitHub and your local machine
    • To delete a branch from your local machine (should only be done AFTER you pull and merge in GitHub), use git checkout main to switch to your main branch, then use git branch -d [branch name] to delete
      • We should delete branches once we have merged to be sure we are never working off of old code. You can always recreate the branch from the updated main if you need to make further changes
    • Once you have merged, you should pull your main again to ensure all changes are up to date using git pull origin main
  • Once you have pulled the branch changes to GitHub using the first two steps above, you should create a pull request
    • To do this, first go to the GitHub repository in your browser
    • Then switch to the branch you want to pull from the branch drop down menu
    • Then click "Compare & pull request"
    • Then, give a detailed yet concise description of what was changed, then click create pull request
    • In most professional and/or collaborative settings, this would now go a colleague or superior for review
    • Once you create the pull request (and if needed, it is approved by someone), you can now merge your changes with the main branch using "Merge Pull Request"
    • Again, be sure to update your local repo by using git pull origin main

JavaScript

JavaScript Basics

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites. Using JavaScript, you can power your websites with effects, interactivity, data communication, and much more
  • In web development, JavaScript is the tool that takes a static HTML/CSS webpage and makes it interactive. On almost every website on the web, every time you do something with a webpage, such as click a button, retrieve data, or fill out a form, JavaScript is at work!
  • Some basics JavaScript features include: variables, operators, conditionals, loops, functions, and events
  • JavaScript runs using the control flow, or the order in which a computer executes code in a JavaScript file
    • Control flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page
  • A semicolon indicates the end of statement and is only required when you need separate statements on a single line. For example: let x = 0; x < topics.length; x++
    • Some people believe it is good practice to put a semicolon at the end of every line. However, this is only partially correct as there are some instances where you should NOT put a semicolon
      • The biggest example where you should not use a semicolon is when you are using curly brackets { } (ie for an if/else statement)
      • The only time you can use a semicolon after a curly bracket is if you are using it for an assignment statement such as var obj {};
  • Comments are made like most other programming languages, using //comment for a single line comment, or 
    /* 
comment
*/
    for long comments
  • All data types except Object define immutable values represented directly at the lowest level of the language. We refer to values of these types as primitive values
  • In computer science, an object is a value in memory which is possibly referenced by an identifier. In JavaScript, objects are the only mutable values. Functions are, in fact, also objects with the additional capability of being callable
  • Most features in JavaScript take arguments, which will be placed in between the () when calling a feature (usually a conditional, loop, function, or event)
  • To link a JavaScript file to your HTML file, always ensure you put it at the very end of your <body> element, typically after the footer. You will use <script src=”source url”></script> to insert it
    • The reason you need to put the JavaScript link at the end of the HTML file is that the script will only affect elements that have already loaded. If you put the script at the beginning, no elements would have loaded, so there would be nothing for the script to affect!
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number

Variables

  • In coding parlance, a variable is simply a named container that allows us to store data in our code
  • In older JavaScript versions, we would establish a variable using var. This would technically still work, but is considered obsolete and not best practice with newer versions of JavaScript
  • In newer JavaScript versions, the way we would define a variable would depend on if we want the variable to be able to be changed (mutable) or unchangeable (immutable)
  • Immutable variables are stored with a const keyword
    • For example const myVariable = ”Steve”;
  • Mutable variables are stored with a let keyword
    • For example let myVariable; which would establish the variable without declaring it
  • To declare a mutable (let) variable, you can initally do this in one of two ways:
    • In the same line that you established the variable as shown with the const example above; for example: let myVariable = "Steve";
    • You can also declare it in a separate line, such as myVariable =”Steve”;. Notice that there is no let in this code; that is because we would have already established the variable earlier in the code in this case as shown above
    • In addition to these examples, you can redeclare a let variable later in the script by using the second example with a different declaration
  • To call (retrieve) your variable, you simply use its name. For example: myVariable;
  • There are many different data types for variables
    • String
      • This is a sequence of text known as a string. To signify that the value is a string, enclose it in single or double quote marks
      • Examples of this would be: let myVariable = 'Steve'; or let myVariable = "Steve";
    • Number
      • This is a number. Numbers don't have quotes around them
      • An example of this would be: let myVariable = 10;
    • Boolean
      • This is a True/False value. The words true and false are special keywords that don't need quote marks
      • An example of this would be: let myVariable = true;
    • Array
      • This is a structure that allows you to store multiple values in a single reference
      • Examples of this would be: let myVariable = [1,'John','Steve',true];. Refer to each member of the array like this: myVariable[0], myVariable[1] etc
    • Object
      • This can be anything. Everything in JavaScript is an object and can be stored in a variable. Keep this in mind as you learn
      • Examples of this would be: let myVariable = document.querySelector('h1');. All of the examples above are technically objects as well

Operators

  • An operator is a mathematical symbol that produces a result based on two values (or variables)
  • Types of operators
    • Addition (Symbol: +)
      • Add two numbers together or combine two strings
      • Examples of this would be: 6 + 9; 'Hello' + 'world!';
    • Subtraction, multiplication, division (Symbol: -, *, /)
      • These do what you'd expect them to do in basic math
      • Examples of this would be: 9 - 3;, 8 * 2; (multiply in JavaScript is an asterisk), 9 / 3;
    • Assignment (Symbol: =)
      • This assigns a value to a variable
      • An example of this would be: let myVariable = 'Steve';
    • Strict equality (Symbol: ===)
      • This performs a test to see if two values are equal and of the same data type. It returns a true/false (Boolean) result
      • An example of this would be: let myVariable = 3; myVariable === 4; (which would then return a false)
    • Not; Does-Not-Equal (Symbol: !, !==)
      • This returns the logically opposite value of what it precedes. It turns a true into a false, etc... When it is used alongside the Equality operator, the negation operator tests whether two values are not equal
      • Examples of this would be: For "Not", the basic expression is true, but the comparison returns false because we negate it:
        let myVariable = 3; !(myVariable === 3);
        "Does-not-equal" gives basically the same result with different syntax. Here we are testing "is myVariable NOT equal to 3". This returns false because myVariable IS equal to 3:
        let myVariable = 3; myVariable !== 3;
  • Mixing data types can lead to some strange results when performing calculations. Be careful that you are referring to your variables correctly, and getting the results you expect. For example, enter '35' + '25' into your console. Why don't you get the result you expected? Because the quote marks turn the numbers into strings, so you've ended up concatenating strings rather than adding numbers. If you enter 35 + 25 you'll get the total of the two numbers

Conditionals

  • Conditionals are code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement
  • An example of this would be:
    • let iceCream = "chocolate";
if (iceCream === "chocolate") {
  alert("Yay, I love chocolate ice cream!");
} else {
  alert("Awwww, but chocolate is my favorite…");
}
  • An if statement is a way of introducing decision-making into our code. Just like its name sounds, when using the if statement, code will only execute if the statement meets a condition, or is true
  • When the computer encounters an if statement in a JavaScript file, it will check to see if the condition that we provide is truthy. If it is, then it will do whatever action we ask it to do in between the curly brackets. If the condition isn’t true or has a value that isn’t truthy, then the computer will ignore whatever is in between the curly brackets and move on
  • What exactly does “truthy” and “falsy” mean in Javascript? A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy

Loops

  • Before we study loops, let's dive deeper into arrays for a moment
    • To create an array, we declare a variable in the same way that we declared a variable with only one data item. However, instead of assigning the variable to a single value, we assign the variable to a group of values
    • As we touched on above, to work with individual data items inside an array, we need to have one more piece of information: the location of the data item in the array. This location is identified with a unique number called an index. Using each piece of data's index, or location number, we can access individual data inside the array
    • Again, as touched on above, To access data from the array, you would use an array index. These would always start with 0. So, for example, if we wanted to pull the 1 from myVariable;, we would use the variable call myVariable[0];
    • Manually changing the values of the index is still not the most efficient way to work with all of the values in the array. What if we wanted to log all of the values of the array individually? We would have to write out a console.log for every item! Imagine how tedious it would be to write this code manually if we had an array that contained 100 or even 1,000 items! In this case, the most efficient way to do this is to use loops, of which there are several types
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. Developers must use another statement, such as the conditional if statement that we just learned, to interrupt the control flow by executing a block of code a specific number of times
    • An example would be: 
      const shapes = ["triangle", "square", "pentagon", "circle"];
for(let x = 0; x < shapes.length; x++) {
  console.log(shapes[x]);
}
  • There are other types of loops as well, and they can be used for other data types and features than arrays, but this is a great starting point to understand loops

Functions

  • Functions are a way of packaging functionality that you wish to reuse. It's possible to define a body of code as a function that executes when you call the function name in your code. This is a good alternative to repeatedly writing the same code
  • A function is different from a for loop or conditional statement because functions do not automatically execute when the JavaScript file is run
  • You must call a function by name in order to execute it
  • Being able to call a function to execute it also means you can control where in the control flow a task is executed
  • The 2 requirements of a JavaScript Function:
    • 1. Define the function
    • 2. Call the function
  • Functions often take arguments: bits of data they need to do their job. Arguments go inside the parentheses (), separated by commas if there is more than one argument
  • Always remember to include the () after the function name (definition). You can leave it empty or give it arguments, but is must always be included
  • You can also define your own functions by defining a name for your function, and then giving it arguments to run
    • This is done by using function functionName(//arguments) { //function action }
    • An example would be: 
      function multiply(num1, num2) {
let result = num1 * num2;
  return result;
}
    • This would then be executed in the script by calling the function; an example of that would be: multiply(4,7);
  • When we declare a function in our JavaScript code, we are creating a reusable block of code that performs a specific task. However, declaring a function does not mean that the code will automatically execute. We need to call the function in the code at the spot we want it to execute
  • It is also important to remember that since control flow dictates that JavaScript files are executed from top to bottom, order matters

Events

  • Real interactivity on a website requires event handlers. These are code structures that listen for activity in the browser and run code in response. The most obvious example is handling the click event, which is fired by the browser when you click on something with your mouse
  • An example of an event handler is: 
    document.querySelector("html").addEventListener("click", function () {
  alert("Ouch! Stop poking me!");
});
  • There are a number of ways to attach an event handler to an element. In the example above, we select the <html> element. We then call its addEventListener() function, passing in the name of the event to listen to ('click') and a function to run when the event happens
  • The function we just passed to addEventListener() here is called an anonymous function, because it doesn't have a name. There's an alternative way of writing anonymous functions, which we call an arrow function. An arrow function uses () => instead of function ()
    • An example of an arrow function would be 
      document.querySelector("html").addEventListener("click", () => {
  alert("Ouch! Stop poking me!");
});

Console

  • The word console refers to a test environment that developers use to check out their code or look for errors/bugs
  • The console will be in your browser. For example, in Chrome, it is under Development Tools
  • We can use the console.log(); method to output a message to the web console by adding an argument
  • In the console.log(); command, .log(); is a method, which means a set of instructions that can be executed by a computer. The .log(); method will output whatever we add inside the parentheses to the console
